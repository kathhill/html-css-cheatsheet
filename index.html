Grid
Property Applied to: Description Values
display: grid Container/parent Defines a Container/parent as a CSS grid. grid is the value
grid-template Container/parent measures rows and columns row measurement/column measurement
gap Container/parent Space between rows and columns (i.e. gutter) row-gap measure column-gap measure (e.g. gap: 20px 5px)
grid-row-start | grid-column-start Child Grid line that the item starts on A number, e.g. 3
grid-row-end | grid-column-end Child Grid line that the same item ends on a number, e.g. 5
grid-row | grid-column Child SHORTHAND for grid-row-start and grid-row-end | grid-column-start and grid-column-end two numbers, e.g. 3 / 5
grid-area Child SHORTHAND for grid-row and grid-column together. Use z-index for overlapping (images etc.) grid-row-start / grid-column-start / grid-row-end / grid-column-end
grid-template-areas Container/parent Allows the naming of sections of a page to use as values for grid-row and grid-column and grid-area e.g. "header header" "nav nav" "info services" "footer footer"
justify-items Container/parent Positions grid items along the x or row axis. start, end, center, stretch
justify-content Container/parent Positions the entire grid along the x or row axis. start, end, center, stretch, space-around, space-between, space-evenly
justify-self Child Specifies how an individual element should position itself with respect to the x or row axis. Overrides justify-items. start, end, center, stretch
align-items Container/parent Positions items along the y or column axis, top to bottom start, end, center, stretch
align-content Container/parent Positions the rows along the y or column axis, from top to bottom start, end, center, stretch, space-around, space-between, space-evenly
align-self Child Specifies how an individual element should position itself with respect to the y or column axis. Overrides align-items. start, end, center, stretch
Implicit vs. Explicit Grid N/A Implicit grid is an algorithm built into the css grid spec that determines default behavior for the placement of elements when there are more than fit into the grid specified by the CSS. Default behavior: items fill rows first, adding new rows as necessary. New rows will only be tall enought to contain the content within theml
grid-auto-rows Container/parent Specifies the size of grid tracks added implicitly (rows in this case). px, %, fr, repeat()
grid-auto-columns Container/parent Specifies the size of grid tracks added implicitly (columns in this case). px, %, fr, repeat()
grid-auto-flow Container/parent In addition to setting the dimensions of implicitly-added rows and columns, we can specify the order in which they are rendered. Specifies whether new elements should be added to rows or columns row, column, dense(pairs with row or column and invokes an algorithm that attempts to fill holes earlier in the grid layout if smaller elements are added
FLEXBOX
Property Applied to: Description Values
display: flex Container/parent Changes an element to a block-level Container/parent with flex items inside of it. flex is the value
display: inline-flex Container/parent allows multiple flex Container/parents to appear inline with each other. inline-flex is the value
justify-content Container/parent is used to space items along the main axis. flex-start (default) | flex-end | center | space-between | space-around
align-items Container/parent is used to space items along the cross axis. stretch (default) | flex-start | flex-end | center | baseline
flex-grow Child is used to specify how much space (and in what proportions) flex items absorb along the main axis. Must be a number | default is 0
flex-shrink child is used to specify how much flex items shrink and in what proportions along the main axis. Must be a number | default is 0
flex-basis Child is used to specify the initial size of an element styled with flex-grow and/or flex-shrink. A width in length, percentage, auto, max-content, min-content, fit-content
flex Child SHORTHAND: is used to specify flex-grow, flex-shrink, and flex-basis in one declaration. flex-grow flex-shrink flex-basis
flex-wrap Container/parent specifies that elements should shift along the cross axis if the flex Container/parent is not large enough. wrap | nowrap (default) | wrap-reverse
align-content Container/parent is used to space rows along the cross axis. space-between | space-around | stretch (default) | center | flex-start | flex-end
flex-direction Container/parent is used to specify the main and cross axes. column | column-reverse | row (default) | row-reverse
flex-flow Container/parent SHORTHAND: is used to specify flex-wrap and flex-direction in one declaration. flex-direction flex-wrap
order Child Specifies the order of the flex items. Must be a number | default value is 0
align-self Child Overrides the defaulot alignment set by the Container/parent's align-items property. center | flex-start | flex-end
Nested flex Container/parents Container/parent Flex Container/parents can be nested inside of each other by declaring display: flex or display: inline-flex for children of flex Container/parents.
